## 场景

维护一个技术公众号，每周需要发布 3-5 篇文章。纯人工运营的痛点：

1. **选题难** — 每天刷各个信息源找热点
2. **写文慢** — 一篇技术文章从构思到发布需要 4-6 小时
3. **排版耗时** — 公众号排版比写代码复杂

目标是：**做一个公众号运营助手，从选题、撰稿到排版全流程提效**。

---

## 实现

### 架构

```
RSS 订阅源 → Coze Bot → ① 每日摘要：筛选热点话题
                     → ② 生成初稿：基于筛选结果写文章
                     → ③ 格式转换：适配公众号排版
                     → ④ 人工审核 → 发布
```

### 核心配置

**第一步：搭建"选题助手"Bot**

```yaml
name: 选题助手
prompt: |
  你是一个技术媒体编辑。每天早上 8 点，你会收到 RSS 摘要列表。

  任务：
  1. 阅读所有文章标题和摘要
  2. 筛选出和 AI/编程/开发者相关的热点
  3. 按热度排序，给出 TOP5 选题建议
  4. 每个选题标注：
     - 热度指数（1-10）
     - 目标读者群
     - 我们的独特角度（和别人写的不一样的切入点）

  RSS 来源：InfoQ、机器之心、36Kr、Hacker News 精选
```

**第二步：搭建"撰稿助手"Bot**

```yaml
name: 撰稿助手
prompt: |
  你是一个技术作者。根据选题和参考资料撰写公众号文章。

  格式要求：
  1. 标题：吸引点击但不标题党
  2. 开头：一句话说清楚读者能获得什么
  3. 正文：2-3 个小标题分段，每段 300-500 字
  4. 代码块：如有代码，用规范格式展示
  5. 结尾：总结 + 引导关注/转发

  风格：通俗但不失深度，像朋友在聊天而不是教授在讲课。
```

**第三步：接入公众号**

Coze 提供了 Bot 发布渠道。关键配置：

- **定时任务**：每天早上 8 点执行选题流程
- **人工确认**：生成的文章先发送到飞书/企业微信审核
- **排版适配**：Coze 输出 Markdown，用工具（md2wechat）转为公众号格式

### 实际流程（一个典型的工作日）

```
08:00  Coze 自动抓取 RSS → 生成 TOP5 选题 → 推送到飞书
08:30  人工选定 1 个选题 → 回复给 Coze
08:35  Coze 生成初稿（含标题 + 正文 + 配图建议）
09:00  人工修改润色
09:30  排版 + 预览
10:00  发布
```

从选题到发布，**总人工时间从 4 小时压缩到约 1 小时**。

## 效果

运营一个 2 万粉的技术公众号，运行两个月：

| 指标 | 变化 |
|------|------|
| 周发文量 | 3 篇 → 5 篇 |
| 单篇写作时间 | 4h → 1h |
| 平均阅读量 | 1200 → 1800 |
| 选题采纳率 | 60%（人工觉得可用的选题比例） |

**翻车案例**：有一次 Agent 写了一篇标题为《震惊！AI 即将取代程序员》的文章，被人工拦截。之后在 prompt 里加了"禁止标题党"。

---

## 要点

1. **人工审核不能省** — Agent 写的内容有 70-80% 可用性，但剩下的 20% 可能包含事实错误或不当表述。现在生成的内容都必须经人审核才能发布。

2. **选题是价值最高的环节** — Agent 的最大贡献不是"写"，而是"筛"。每天自动整理热点，节省了刷信息流的时间。

3. **风格需要调教** — 初期写出来的文章"AI 味"很重（太正式、缺幽默感）。通过在 prompt 里加"想象你在和同事喝咖啡聊天"类描述可以改善。

4. **配图方案** — 目前选 Unsplash 免费图 + 自动截图逻辑。封面图可以用 DALL-E 生成。

---

## 扩展思路

- 自动生成多平台适配（公众号 → 知乎 → 掘金 → Twitter）
- 文章数据复盘：打开率、完读率、转发率分析
- 接入热点监控：监测 GitHub Trending、知乎热榜、微博热搜
