## 场景

电商团队每天需要处理 2000+ 客服咨询，其中约 60% 是重复性问题：物流查询、退换货政策、尺码推荐。人工客服疲于应对，平均响应时间超过 15 分钟。

目标是：**让 AI Agent 处理常规咨询，人工客服只处理复杂问题**。

---

## 实现

### 架构

```
用户提问 → Dify Agent → 意图分类 → ① 常规问题 → 知识库 RAG → 自动回复
                                   → ② 复杂问题 → 转接人工
                                   → ③ 售后问题 → 调用 ERP API 查询订单
```

### 核心配置

**第一步：知识库准备**

在 Dify 中创建知识库，导入以下文档：

- 常见问题 FAQ（30+ 条）
- 退换货政策
- 物流合作商及配送时效
- 尺码对照表
- 商品使用说明

**第二步：Agent 提示词**

```yaml
name: 客服助手
model: DeepSeek V3
prompt: |
  你是电商客服助手助手。你的职责是回答用户关于商品、订单、物流、售后的问题。

  规则：
  1. 先判断用户意图：查询/投诉/售后/闲聊
  2. 查询类 → 从知识库检索答案
  3. 售后类 → 调用订单查询工具获取订单状态
  4. 无法回答 → 转接人工客服
  5. 回复简洁，不超过 100 字
  6. 不要编造信息，不确定就说"帮我转接人工客服"

tools:
  - type: dataset_retrieval
    name: 商品知识库
  - type: api_call
    name: 查询订单
    url: https://api.example.com/order/{order_id}
```

**第三步：转人工策略**

设置触发转人工的条件：

```
- 用户情绪激烈（检测到辱骂/多次追问）
- Agent 连续两次回答"不确定"
- 用户主动要求"转人工"
- 涉及退款金额超过 500 元
```

对接飞书/企业微信群，转人工时自动推送上下文摘要。

## 效果

在一家日订单 500+ 的电商店铺运行一个月：

| 指标 | 变化 |
|------|------|
| 自动回复占比 | 58% |
| 平均响应时间 | 15min → 45s |
| 人工客服工单量 | 减少 52% |
| 用户满意度 | 91%（和纯人工持平） |
| Agent 转人工准确率 | 85% |

**典型数据**：用户提问后 45 秒内得到答复，其中近 6 成不需要人工介入。

---

## 要点

1. **知识库质量决定上限** — 模糊/矛盾的文档会让 Agent 给出错误答案。投入时间把 FAQ 写好，比调 prompt 效果大 10 倍。

2. **意图分类是命门** — Dify 内置的分类不够细，建议用 few-shot 方式给每个意图配 3-5 个示例问题。

3. **转人工要做干净** — 转人工时携带完整的对话上下文和 Agent 已尝试的方案，让人工客服无缝接管。

4. **冷启动** — 最初两周建议保持在"建议模式"（Agent 给出回复建议，人工确认后发出），收集足够数据后再切自动。
